ONE-PAGE LANGUAGE GUIDE

--------------------------------------------------
PURPOSE
--------------------------------------------------
GDBL is a custom behavior-definition language for drones.
Users define WHAT the drone should attempt to do,
never HOW the drone flies.

The platform enforces non-bypassable safety,
human override, and full explainability.

--------------------------------------------------
CORE SAFETY RULES (NON-NEGOTIABLE)
--------------------------------------------------
- No motor control
- No speed, altitude, or direction control
- No navigation paths
- No loops, functions, or timers
- No writable variables
- No safety bypass
- All actions go through a safety kernel
- All decisions are logged and explainable

--------------------------------------------------
PROGRAM STRUCTURE (MANDATORY)
--------------------------------------------------
Every program must contain EXACTLY ONE rule.

RULE <rule_name>
WHEN <event>
IF <variable> <operator> <value>   (optional)
DO <action>

- Keywords must be UPPERCASE
- Order of lines is mandatory
- Only one action is allowed

--------------------------------------------------
RESERVED KEYWORDS
--------------------------------------------------
RULE
WHEN
IF
DO

--------------------------------------------------
EVENTS (WHEN)
--------------------------------------------------
Events are generated by sensors or AI.

Supported events:
- anomaly_detected     : AI detected abnormal situation
- obstacle_detected    : Obstacle detected
- battery_low          : Battery below safe threshold
- gps_lost             : GPS signal unavailable

Example:
WHEN anomaly_detected

--------------------------------------------------
VARIABLES / SIGNALS (READ-ONLY)
--------------------------------------------------
Variables are system-provided signals.
Users cannot create or modify variables.

Supported variables:
- confidence (float, range 0.0 to 1.0)

Example:
IF confidence > 0.7

--------------------------------------------------
OPERATORS
--------------------------------------------------
Supported operators:
>   <   >=   <=   ==

Unsupported operators:
!=   &&   ||   >>

--------------------------------------------------
ACTIONS (DO)
--------------------------------------------------
Actions represent high-level behavior intent.

Allowed actions:
- HOVER        : Hold position safely
- STOP         : Stop mission execution
- RETURN_HOME  : Return to launch point
- ABORT        : Emergency abort

Example:
DO HOVER

--------------------------------------------------
FORBIDDEN ACTIONS (COMPILE-TIME ERROR)
--------------------------------------------------
The following actions are NOT allowed:

SET_SPEED
SET_ALTITUDE
MOVE_FORWARD
TURN_LEFT
MOTOR_ON
NAVIGATE_TO

Using any of these will cause a compiler error.

--------------------------------------------------
VALID EXAMPLES
--------------------------------------------------

Example 1: Risk avoidance
RULE avoid_risk
WHEN anomaly_detected
IF confidence > 0.7
DO HOVER

Example 2: Battery safety
RULE battery_guard
WHEN battery_low
DO RETURN_HOME

Example 3: GPS failure
RULE gps_fail
WHEN gps_lost
DO ABORT

--------------------------------------------------
INVALID EXAMPLES
--------------------------------------------------

Unsafe control:
RULE unsafe
WHEN anomaly_detected
DO SET_SPEED

Missing rule declaration:
WHEN anomaly_detected
DO HOVER

Invalid operator:
RULE bad_if
WHEN anomaly_detected
IF confidence >> 0.7
DO HOVER

--------------------------------------------------
RUNTIME BEHAVIOR
--------------------------------------------------
Even valid rules may be overridden or rejected at runtime.

Execution flow:
1. AI generates signals
2. Rule produces behavior intent
3. Safety kernel evaluates intent
4. Final action is APPROVE, OVERRIDE, or REJECT
5. Decision is logged with reason

Example log:
INTENT=HOVER | DECISION=OVERRIDE | ACTION=RETURN_HOME | REASON=Battery unsafe

--------------------------------------------------
SAFETY GUARANTEES
--------------------------------------------------
- No flight control exposure
- No motor access
- Mandatory human override
- Compile-time and runtime safety enforcement
- Fully explainable decisions

--------------------------------------------------
MENTAL MODEL
--------------------------------------------------
GDBL is a BEHAVIOR DECLARATION LANGUAGE,
not a flight control or scripting language.

--------------------------------------------------
ONE-LINE SUMMARY
--------------------------------------------------
GDBL enables safe, explainable programming of drone behavior
with non-bypassable safety governance.
